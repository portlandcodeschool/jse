JavaScript Fundamentals

Modeling the World

Objects


var personName = "Whitney";
var personAge = 4;
var personFavoriteColor = "blue";
var personHobby = "cooking";

These variables are related, but only because we named them that way.


var person = {
  name: "Whitney",
  age: 4,
  favoriteColor: "blue",
  hobby: "cooking"
};

Objects allow us to really group things together.

Note that there's no comma on the last line.

Accessing properties of objects is easy.


console.log(person.name);
console.log(person.age);

person.age = 12;
person.favoriteColor = "orange";
console.log(person);

And there are actually two ways to access properties.


console.log(person['name']);
console.log(person['age']);

person['age'] = 12;
person['favoriteColor'] = "orange";
console.log(person);

For now we're going to use the first way.

Dot notation is preferred unless your property name includes a symbol that is not allowed via dot notation (like a dot).

Learning to Speak


console.log(person1.name + "says \"dada\" for the first time.");
console.log(person2.name + "says \"dada\" for the first time.");

Anyone can speak, but now we're repeating ourselves.

Functions

Functions are chunks of code that can be reused.

Function Anatomy


var speak = function(person) {
  console.log(person.name + "says \"dada\" for the first time.");
};
This is read: I'm declaring the variable speak to be a function that takes one argument, a person.

Using Functions


var speak = function(person) {
  console.log(person.name + "says \"dada\" for the first time.");
};
speak(person1);
speak(person2);

You say that you're calling the function speak.
